# 执行文档导航

## 1. 文档说明

本目录用于承接 `README.md` 第 17 节“给后续 AI 助理的执行要求”，将需求规格说明书进一步拆解为可执行的技术与开发文档，供后续 AI 助理或开发团队按顺序推进。

## 2. 文档清单

### 2.1 `01-system-architecture.md`

用途：

1. 定义系统边界、模块划分、技术选型建议。
2. 说明前后端、AI 解经、内容审核、音频播放等核心链路如何协作。
3. 作为后续数据库、接口与任务拆解的总依据。

### 2.2 `02-database-design.md`

用途：

1. 定义核心数据实体、表结构建议、主外键关系。
2. 说明关键字段、状态字段、索引建议和约束。
3. 作为后续 ORM 建模、迁移脚本与后端开发依据。

### 2.3 `03-api-spec.md`

用途：

1. 定义 MVP 阶段主要接口清单。
2. 统一请求方式、鉴权要求、输入输出结构与错误码思路。
3. 作为前后端联调和接口开发依据。

### 2.4 `04-mvp-task-breakdown.md`

用途：

1. 将 MVP 功能拆成可实现、可测试、可交付的开发任务。
2. 按优先级和依赖关系排序。
3. 作为开发排期、AI 编码执行和验收跟踪依据。

### 2.5 `05-implementation-roadmap.md`

用途：

1. 将 `README.md` 第 17 节的执行步骤转换为阶段性推进路线。
2. 明确每个阶段的输入、输出、风险点和完成标准。
3. 作为整体实施顺序和里程碑依据。

## 3. 推荐阅读顺序

1. 先读 `README.md`
2. 再读 `docs/01-system-architecture.md`
3. 然后读 `docs/02-database-design.md`
4. 接着读 `docs/03-api-spec.md`
5. 再读 `docs/04-mvp-task-breakdown.md`
6. 最后读 `docs/05-implementation-roadmap.md`

## 4. 执行顺序建议

1. 先完成基础项目初始化和模块骨架。
2. 优先打通账号系统。
3. 然后打通随机经文生成与经文展示。
4. 再打通“解经”内容链路。
5. 之后完成标注、感悟、公开交流、私信触发、赞美播放。
6. 最后补齐后台、审核、安全、性能和视觉打磨。

## 5. 与 `README.md` 的关系

1. `README.md` 是产品需求总说明。
2. `docs` 目录中的文档是技术落地与执行拆解。
3. 若 `README.md` 与本目录文档有冲突，应优先更新 `README.md` 后再同步修订本目录文档。
