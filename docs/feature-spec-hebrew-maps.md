# 功能需求规格说明书：希伯来语发音音频与圣经地图

## 1. 文档目的

本文档针对以下两个新增功能进行规范化定义：

1. 希伯来语发音音频播放
2. 圣经地图（Bible Maps）

供开发、测试与验收使用。

---

## 2. 功能一：希伯来语发音音频播放

### 2.1 需求背景

解经内容中经常出现希伯来原文词汇（如 בְּרֵאשִׁית、יְהוֹשׁוּעַ、שָׁלוֹם 等）。当前系统已提供拉丁音译标注，但用户希望进一步通过点击小喇叭图标直接收听希伯来语发音，以辅助原文学习。

### 2.2 功能目标

- 在展示希伯来文的组件中提供可点击的发音按钮。
- 点击后使用浏览器原生语音合成能力朗读该希伯来词汇。
- 对不支持的浏览器给出温和提示。
- 保持界面简洁，不破坏现有 Ruby 音译标注的排版。

### 2.3 使用场景

1. 用户阅读解经内容时，看到希伯来原文词汇及音译。
2. 用户点击词汇旁的小喇叭图标。
3. 浏览器朗读该希伯来词汇。
4. 朗读期间图标显示播放状态，朗读结束后恢复。

### 2.4 功能需求

#### 2.4.1 发音触发

1. 每个希伯来文词组旁显示一个“小喇叭”图标按钮。
2. 按钮默认状态为可点击。
3. 点击后调用浏览器 Web Speech API 朗读该希伯来文。
4. 朗读语言设置为 `he-IL`（希伯来语，以色列）。
5. 朗读内容为原始希伯来字符串（保留元音符号，便于 TTS 引擎正确发音）。

#### 2.4.2 状态反馈

1. 朗读开始时，图标变为“播放中”状态（例如声波动画或高亮）。
2. 朗读结束、被取消或出错时，图标恢复默认状态。
3. 若浏览器不支持语音合成，点击时提示“当前浏览器暂不支持希伯来语朗读”。

#### 2.4.3 边界情况

1. 连续点击多个词汇时，应先取消当前朗读，再开始新的朗读。
2. 页面切换或组件卸载时，应停止当前朗读。
3. 希伯来文为空或仅包含标点时，不显示喇叭按钮。

### 2.5 技术实现要点

- 使用 `window.speechSynthesis` 与 `SpeechSynthesisUtterance`。
- 设置 `utterance.lang = 'he-IL'`。
- 使用 `utterance.onend` / `utterance.onerror` 管理状态。
- 图标使用 SVG，避免引入额外图标库。

## 3. 功能二：圣经地图

### 3.1 需求背景

圣经历史涉及大量地理信息，用户在研读经文时，若能直观查看人物行进路线、地名位置及相关经文背景，将有助于理解经文的时空背景。

### 3.2 功能目标

- 提供一个独立的“圣经地图”菜单入口。
- 在交互式地图上展示圣经中的重要地点。
- 支持搜索地名并定位。
- 支持地图缩放与平移。
- 提供多条经典圣经路线供用户选择查看。
- 每个地点提供注解，点击后可查看相关经文与历史背景介绍。

### 3.3 使用场景

1. 用户从导航栏点击“圣经地图”。
2. 页面加载交互式地图，默认展示某条经典路线（如保罗第一次宣教旅程）。
3. 用户可在搜索框输入地名（如“耶路撒冷”），地图自动定位到该地点。
4. 用户点击地图上的标记，弹出地点名称、相关经文与历史背景。
5. 用户可通过下拉菜单切换不同路线。
6. 用户可自由缩放、平移地图查看细节。

### 3.4 功能需求

#### 3.4.1 入口与页面

1. 在顶部导航栏新增“圣经地图”入口。
2. 点击后跳转至 `/maps` 页面。
3. 页面标题为“圣经地图”。

#### 3.4.2 地图展示

1. 使用 Leaflet 开源地图库渲染交互式地图。
2. 默认视图覆盖以色列、巴勒斯坦、约旦、埃及、土耳其、希腊等圣经核心区域。
3. 地图支持鼠标滚轮缩放、拖拽平移、双击放大。
4. 在移动端支持手势缩放。

#### 3.4.3 路线选择

1. 提供路线选择器，至少包含以下路线：
   - 保罗第一次宣教旅程
   - 保罗第二次宣教旅程
   - 保罗第三次宣教旅程
   - 摩西出埃及路线
   - 亚伯拉罕迁往迦南路线
2. 选择路线后，地图自动调整视野以展示整条路线。
3. 路线用带箭头的折线表示，按行程顺序连接各地点。
4. 不同路线使用不同颜色区分。

#### 3.4.4 地点标记与搜索

1. 每条路线上的关键地点用标记点（Marker）表示。
2. 标记点按经过顺序编号或采用不同样式。
3. 提供搜索框，用户输入地名后，地图定位到最近匹配地点并打开注解。
4. 搜索支持中文与圣经地名对照。

#### 3.4.5 地点注解

1. 点击标记弹出信息卡片，内容包括：
   - 地点名称（中文 + 原文/英文名）
   - 地理位置说明
   - 相关经文引用
   - 历史背景与圣经意义简介
2. 经文引用格式统一，如“使徒行传 13:4-5”。
3. 背景介绍需符合基督教圣经知识，简明准确。

### 3.5 路线与地点数据规范

#### 3.5.1 数据准确性要求

1. 所有地名坐标采用学术界或主流圣经地图常用的近似坐标。
2. 对存在学术争议的地点（如西奈山、过红海位置），在介绍中标注“传统认为”或“约位于”。
3. 路线顺序需与使徒行传、出埃及记、创世记等经文记载一致。
4. 经文引用使用简体中文和合本格式。

#### 3.5.2 路线定义

详见 [`/workspace/frontend/src/data/bibleMaps.ts`](/workspace/frontend/src/data/bibleMaps.ts)。

### 3.6 界面要求

1. 地图占据页面主要区域，清晰、无冗余控件。
2. 路线选择器与搜索框位于地图左上角或顶部，采用半透明卡片样式。
3. 标记点与路线颜色在地图上清晰可辨。
4. 信息卡片宽度适中，不影响地图主体查看。
5. 整体风格与现有应用一致（温暖、庄重、简洁）。

### 3.7 技术实现要点

- 使用 `leaflet` + `react-leaflet` 实现地图。
- 地图瓦片使用 OpenStreetMap（默认）或 CartoDB Voyager（清晰、低饱和，更适合圣经地图氛围）。
- 路线数据以 TypeScript 对象形式静态维护，便于扩展。
- 地点 Popup 使用 Leaflet 原生 Popup 或自定义 React 组件。

---

## 4. 非功能需求

### 4.1 性能

1. 地图页面首次加载时间控制在 3 秒内。
2. 路线切换、搜索定位响应流畅，无明显卡顿。

### 4.2 兼容性

1. 希伯来语朗读功能在支持 Web Speech API 的现代浏览器中可用。
2. 地图功能支持桌面端与主流移动端浏览器。

### 4.3 可维护性

1. 地图数据与渲染逻辑分离，便于后续新增路线或地点。
2. 希伯来语朗读逻辑封装为独立 Hook，便于复用。

## 5. 验收标准

### 5.1 希伯来语发音

- [ ] 解经页面中的希伯来文旁出现小喇叭图标。
- [ ] 点击图标后浏览器能朗读希伯来语。
- [ ] 不支持语音合成的浏览器给出明确提示。
- [ ] 连续点击不同词汇时，前一次朗读被正确取消。

### 5.2 圣经地图

- [ ] 导航栏出现“圣经地图”入口。
- [ ] `/maps` 页面正常加载交互式地图。
- [ ] 可切换至少 5 条经典路线。
- [ ] 路线在地图上正确绘制并连接各地点。
- [ ] 搜索地名后地图定位并打开注解。
- [ ] 点击标记可查看地点名称、经文引用与历史背景。
- [ ] 地图支持缩放与平移。

---

## 6. 给后续开发者的说明

1. 圣经地图中的地点坐标均为近似值，主要用于辅助读经，不作为考古或神学定论。
2. 如需新增路线，请在 `bibleMaps.ts` 中按现有数据结构扩展即可。
3. 希伯来语朗读依赖浏览器 TTS 引擎，不同浏览器发音质量可能不同。
